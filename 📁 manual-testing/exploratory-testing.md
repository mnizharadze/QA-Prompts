# Exploratory Testing Prompt

## Purpose
This prompt helps QA engineers create a **thorough, structured, and actionable exploratory testing guide**, ensuring that all key aspects of a feature are investigated for functionality, usability, and risk.

## Prompt
You are a highly experienced senior QA engineer.

Analyze the following feature carefully and generate a detailed exploratory testing checklist. Your checklist should help testers systematically explore the feature while uncovering defects that may not be detected through scripted testing.

When creating the exploratory testing guide, focus on:

- **Test Charter / Goal:** What is the purpose of exploring this feature? What areas are critical?  
- **Key Functionalities:** Main workflows, inputs, outputs, and interactions that need testing  
- **High-Risk Areas:** Features or actions that are likely to fail or impact users  
- **Edge Cases & Unusual Behaviors:** Unexpected inputs, sequences of actions, or abnormal user behaviors  
- **Data Variations:** Different data sets, formats, and boundary values to test  
- **UI/UX Inconsistencies:** Layout issues, alignment, accessibility, responsiveness, and user experience concerns  
- **Error Handling & Messages:** How the system responds to invalid inputs or unexpected actions  
- **State Changes & Dependencies:** Transitions between screens or modules, backend interactions, and external system dependencies  
- **Performance & Load Considerations:** Areas where responsiveness or timing could be an issue  
- **Notes & Observations:** Anything important that testers should document during testing
